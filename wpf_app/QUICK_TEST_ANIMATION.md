# ⚡ 快速测试动画系统

**5 分钟验证 PixelActor 动画是否正常工作**

---

## 🚀 启动应用

```bash
cd wpf_app/WorkHoursTimer
dotnet run
```

或双击运行：
```bash
cd wpf_app
./build-and-run.bat
```

---

## ✅ 核心测试（3 步）

### 1️⃣ 创建挂件
- 点击主窗口的 **"创建挂件"** 按钮
- **预期**: 挂件窗口出现在屏幕上

### 2️⃣ 验证待机动画
- 观察挂件左侧的勇者角色
- **预期**: 勇者在播放待机动画（呼吸动作，8帧循环）
- **检查点**:
  - ✅ 动画流畅，无闪烁
  - ✅ 像素清晰，无模糊
  - ✅ 循环播放，无卡顿

### 3️⃣ 验证攻击动画
- 在主窗口点击 **"开始工作"** 按钮
- **预期**: 勇者切换到攻击动画（挥剑动作，6帧循环）
- 点击 **"停止"** 按钮
- **预期**: 勇者切换回待机动画

## ❌ 如果出现问题

### 问题 1: 应用崩溃
**检查**: 查看控制台错误信息
**可能原因**: 图片资源未正确嵌入
**解决**: 运行 `dotnet clean` 然后 `dotnet build`

### 问题 2: 动画不播放
**检查**: 勇者是否显示为静态图片
**可能原因**: 帧路径错误或图片加载失败
**解决**: 查看 Debug 输出窗口的错误日志

### 问题 3: 动画模糊
**检查**: 像素是否清晰
**可能原因**: RenderOptions 未设置
**解决**: 已在代码中设置，应该不会出现此问题
